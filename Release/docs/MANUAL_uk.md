# FlexiSoft Runtime — сервісний посібник

**ТЕСТ ЛОКАЛІЗАЦІЇ: MANUAL_uk.md**

## 1. Огляд

FlexiSoft Runtime — це сервісна програма для моніторингу вибраних станів контролерів безпеки SICK Flexi Soft через комунікацію RK512.

Програма призначена для машин, де контролер безпеки може повідомити про помилку після відкриття і закриття захисного кожуха. Зазвичай причина — неодночасне перемикання двоканального магнітного датчика безпеки.

FlexiSoft Runtime виявляє налаштовані помилки входів, інформує оператора і дозволяє надіслати контрольовану команду скидання. Якщо фізичний стан безпеки правильний, скидання може прибрати помилку. Якщо помилка все ще фізично присутня, вона залишається активною і оператор отримує нове попередження.

FlexiSoft Runtime не є контролером безпеки і не замінює жодну функцію безпеки машини.

## 3. Іконка tray

Програма працює в системній області Windows.

| Колір | Значення |
| --- | --- |
| Зелений | Зв’язок OK, немає активної налаштованої помилки. |
| Сірий | Немає активного зв’язку. |
| Жовтий / блимання | Відкриття транспорту або reconnect. |
| Червоний | Принаймні один налаштований канал у помилці. |

## 4. Меню tray

Меню показує стан каналів і основні команди.

`Send CHx command` ставить у чергу reset-команду для вибраного каналу. Команда виконується комунікаційним потоком runtime, а не напряму з UI.

`Reload config` перечитує конфігурацію і перезапускає комунікацію.

`Reconnect` зупиняє поточний транспорт і запускає зв’язок знову.

`Language` змінює активну мову UI.

`About` відкриває діалог з версією, документацією, логом і автором.

## 5. Попередження

### 5.1 Active Input Error Alert

Показується, коли один або кілька налаштованих каналів повідомляють про помилку.

| Кнопка | Дія |
| --- | --- |
| YES | Надіслати одну спільну команду для всіх зазначених помилок. |
| NO | Закрити попередження без команди. |

### 5.2 Repeat Fault Alert

Показується, коли помилка входу повторно повертається після команди.

Можливі причини:

* захист закритий неправильно,
* відсутній один канал двоканального датчика,
* датчик неправильно вирівняний,
* проблема з проводкою,
* помилка реальна і її не можна обходити.

### 5.3 Command Failed Alert

Показується, коли команда не була завершена правильно.

Команда успішна тільки після прийняття ON telegram, закінчення pulse time і прийняття OFF telegram.

## 6. Конфігурація

Основний конфігураційний файл зараз:

```text
config.json
```

Пізніше планується перенесення JSON файлів у `conf/`.

## 7. transport

`transport` вибирає секцію `network` або `serial`.

## 8. serial

Серійний транспорт використовує COM порт, baudrate, parity, stop bits і timeout. Перевірене значення baudrate: `115200`.

## 9. network

Режим `tcp_client` використовує IP адресу і порт TCP-to-RS232 перетворювача.

## 10. rk512

Секція `rk512` задає device id, reply id і token telegram.

## 11. poll

Секція `poll` задає період читання, блок і розмір даних.

## 12. inputs

Кожен input описує один логічний вхід безпеки. Якщо канал увімкнений і `ok_bit` дорівнює false, канал вважається помилковим.

## 13. output

`output` описує command bit для reset-pulse.

## 14. repeat_fault

`repeat_fault` задає кількість повторів, часовe вікно і текст повторної помилки.

## 15. ui

Секція `ui` містить текст tooltip для tray.

## 16. logging

Runtime log завжди пишеться англійською, незалежно від мови UI.

## 17. debug

`force_command_fail` використовується тільки для тестування Command Failed Alert.

## 18. Локалізація

UI тексти завантажуються з `languages.json`.

Документація відкривається з папки `docs/`:

```text
docs\README_<lang>.md -> docs\README.md
docs\MANUAL_<lang>.md -> docs\MANUAL.md
```

Українська використовує suffix `uk`. `ua` не підтримується.

## 19. version.json

`version.json` містить інформацію про продукт, версію, опис, автора, email і дату збірки.

## 20. Усунення проблем

Якщо tray іконка сіра, перевірте порт, TCP перетворювач, IP адресу, живлення Flexi Soft і кабелі.

Якщо іконка червона, принаймні один налаштований канал повідомляє про помилку.

Якщо попередження повертається, фізична помилка ймовірно все ще присутня.

## 21. Примітки з безпеки

FlexiSoft Runtime не є пристроєм безпеки і не повинен використовуватися для обходу функцій безпеки.
