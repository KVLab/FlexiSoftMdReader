# FlexiSoft Runtime — швидкий старт

**ТЕСТ ЛОКАЛІЗАЦІЇ: README_uk.md**

FlexiSoft Runtime — це самостійна сервісна програма для моніторингу контролерів безпеки SICK Flexi Soft, контрольованого відновлення вибраних помилок і сповіщення оператора про повторювані помилки входів безпеки.

## Призначення

Програма призначена для машин, де Flexi Soft повідомляє про помилки входів безпеки через неодночасне перемикання двоканальних датчиків безпеки.

Типовий сценарій:

1. Захисний кожух відкривається і закривається.
2. Обидва канали безпеки перемикаються не зовсім одночасно.
3. Flexi Soft повідомляє про помилку одночасності входу.
4. FlexiSoft Runtime виявляє помилку.
5. Оператор може підтвердити контрольовану команду скидання.
6. Якщо причина була лише у різниці часу, вхід повертається в OK.
7. Якщо фізична помилка все ще присутня, помилка залишається активною і оператор отримує нове попередження.

Програма не замінює функцію безпеки Flexi Soft. Вона тільки виконує моніторинг, повідомляє оператора і надсилає налаштовані команди скидання.

## Основні функції

* Комунікація RK512 зі SICK Flexi Soft.
* Послідовний транспорт.
* TCP client транспорт через TCP-to-RS232 перетворювач.
* Моніторинг до чотирьох налаштованих каналів.
* Одне спільне попередження для кількох активних помилок.
* Контрольована команда скидання для відповідних каналів.
* Виявлення повторної помилки.
* Попередження про невдалу команду з можливістю повтору.
* Індикатор стану в tray.
* Runtime журнал.
* Налаштування через JSON.
* Сумісність від Windows XP SP3 32-bit до Windows 11.

## Перший запуск

1. Скопіюйте `FlexiSoftRuntime.exe` і конфігураційні файли у робочу папку.
2. Перевірте параметри зв’язку у `config.json`.
3. Запустіть `FlexiSoftRuntime.exe`.
4. У системному tray Windows з’явиться іконка.
5. Дочекайтесь, поки іконка стане зеленою.

## Кольори іконки

| Колір | Значення |
| --- | --- |
| Зелений | Зв’язок OK і немає активної налаштованої помилки. |
| Сірий | Зв’язок неактивний або транспорт відключений. |
| Жовтий / блимання | Виконується підключення або reconnect. |
| Червоний | Зв’язок OK, але принаймні один канал у помилці. |

## Попередження

* **Input Error Alert** — активна помилка входу.
* **Repeat Fault Alert** — та сама помилка повторюється після команди.
* **Command Failed Alert** — команда скидання не була виконана правильно.
